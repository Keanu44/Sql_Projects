# BMW Used Car Market Analysis (MotherDuck SQL Project)

## Project Overview

- This project analyzes a real-world dataset of BMW used car listings to uncover pricing dynamics, depreciation patterns, and market behavior using SQL inside a cloud-native analytics environment (MotherDuck).

- Rather than importing files into a traditional staging table, the uploaded dataset in MotherDuck (project.main.bmw) is treated as the source layer, and transformed into a relational analytical model to support business-style exploration.

## Business Questions Answered

**This analysis was designed to answer key automotive market questions:**

What is the overall pricing landscape of BMW vehicles?

Which models dominate the used-car supply?

How does vehicle age influence depreciation?

Does mileage significantly impact resale value?

Do fuel type and transmission affect pricing?

Which BMW models retain value best over time?


## Tech Stack

- Platform: MotherDuck (Cloud DuckDB)

- Language: SQL

- Environment: VS Code + MotherDuck Extension


## **Why This Project Matters**

- This project demonstrates how SQL can be used not just for querying, but for:

- Designing an analytical data model

- Structuring raw datasets into business-ready formats

- Performing exploratory analysis in a cloud data environment

- Simulating real-world pricing intelligence workflows.
